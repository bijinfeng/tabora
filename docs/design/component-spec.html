官网搭建规则

官网不是单独的视觉稿，而是基于 Tabora 基础组件向上组合的营销站点层。站点组件使用 Site* 命名，消费同一组 token，并在内部复用 Button、Input、Kbd、Badge、 Chip、Field、Table 等基础件。

Site* 命名复用基础件响应式网页官网专用组合
基础组件层 Button、Input、Chip、Badge、Kbd、Field、Toast 等可进入插件内容区和官网。
官网组合层 SiteSection、SiteTopnav、SiteHero、SiteProductPreview、SiteProof、SiteDownloadCard、DocsShell、DocsCodeBlock、DocsComponentExample、SiteCTA、SiteWaitlist、SiteFooter。
宿主工作台层 WorkbenchRail、WorkbenchGrid、SettingsHost、Modal 等由 shell/layout host 提供，官网只预览。

组件分层

  • @tabora/ui：Button、Input、Select、Badge、Kbd、Toast 等基础组件。
  • Site*：官网页面结构组件，只负责营销站点的节奏、导航、首屏、产品预览、证明和 CTA。
  • Workbench*：真实工作台宿主组件，不放进官网组件层，也不由官网直接实现业务状态。
官网区域 组件 基础依赖 响应式要求
