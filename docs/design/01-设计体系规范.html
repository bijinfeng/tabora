设计原则

以下五条原则指导 Tabora 中每一个视觉和交互决策。按优先级排序——当两条原则冲突时，更高优先级的胜出。

1. 安静优先于响亮

Tabora 是一个每天打开数十次的工具。界面绝不能索求注意力。色彩、动画和装饰的存在意义是澄清状态和层级——绝不是为了取悦眼球。一次出色的 Tabora 使用体验，是你完全忘记了 UI 的存在。

2. 个性优先于通用

每个用户的工作台都应该有「自己的」感觉。设计语言提供一个安静、中性的地基——用户的卡片、链接、笔记和排列组合来提供个性。我们不强加品牌印记，我们提供画布。

3. 稳定优先于流动

卡片尺寸、位置和交互状态绝不允许引起布局抖动。hover 效果只改变颜色和边框，不改变尺寸。加载状态不推动内容位移。拖拽排序不坍塌网格。稳定铸造信任。

4. 清晰优先于花哨

每个可交互元素必须一眼可扫、一目了然。使用标准交互模式（勾选框表示完成、开关表示启用/禁用、下拉表示列表选择）。绝不发明新的交互范式来解决已被解决过的问题。

5. 默认即无障碍

无障碍性不是在发布前跑一遍的检查清单。每个组件在出厂时就带有正确的 ARIA 角色、键盘导航、焦点管理和色彩对比度。目标为 WCAG 2.2 AA 级合规。

色彩体系

Tabora 使用三层 token 体系（原始 → 语义 → 组件），确保主题可互换而不破坏视觉关系。插件和组件只引用语义 token——绝不硬编码原始色值。

色彩使用规则

强调色占比

  • 绿色强调色在可视面积中的占比必须低于 5%。Tabora 不是"绿色主题"产品。
  • 强调色仅用于：主按钮、侧栏/导航的激活态、焦点环、已勾选态、选中边框和小型数据高亮。
  • 强调色绝不用于：正文文本、卡片背景、装饰元素、大面积页面区域。

文本对比度

  • text 在 surface 上的对比度不低于 4.5:1（WCAG AA 普通文本要求）。
  • text-muted 对比度不低于 3:1（WCAG AA 大文本或 UI 组件要求）。在白色上的实际值约 4.8:1——超出要求。
  • 绝不将 text-subtle 用于用户完成任务所必须阅读的内容。

边框作为层级手段

  • 边框是 Tabora 的主要视觉层级机制——不是阴影，不是背景渐变。
  • 卡片默认为 1px line 边框，无阴影。
  • hover 态：边框切换为 line-strong。不改变尺寸，不缩放变形。
  • 选中态：边框切换为 accent，叠加 accent-soft 背景。

暗色主题色值

暗色主题镜像明亮主题结构。页面底色使用深炭绿，表面使用稍浅的层次。强调色切换为更明亮的翡翠绿以维持暗色背景下的对比度。

字体排印

Tabora 使用单一字体——Inter——以系统无衬线栈作为回退。全局使用紧凑型 productive type scale。不使用展示级或 hero 字号，不使用衬线配对。

间距与网格

Tabora 采用4px 基础单位，2px 步进用于微调。所有间距通过 token 定义——组件中绝不硬编码 px 值。

Token 值 使用场景
--tabora-space-0 0 无边距
--tabora-space-1 2px 微调：紧凑场景中的图标-文本间距
--tabora-space-2 4px 图标-文本小间距、勾选框-标签间距
--tabora-space-3 8px 控件内部紧凑间距、行内间距
--tabora-space-4 12px 卡片内小段落间距、图标行间距
--tabora-space-5 16px 卡片内边距、组件组间距、列表项内边距
--tabora-space-6 20px 中等内容分隔
--tabora-space-7 24px 页面段落间距、网格区域分隔
--tabora-space-8 32px 页面外边距、较大章节分隔
--tabora-space-10 48px 全屏视图大分隔

组件间距 vs 布局间距

  • 组件内部使用 padding。绝不在可复用组件上设置 margin。
  • 布局容器使用 gap（flex/grid gap）分隔子元素。绝不依赖子元素 margin 拼布局。
  • 这确保组件在任何布局上下文中都能工作，无需重置 margin。

动效

Tabora 中的动效是反馈，不是装饰。每个动画都服务于功能性目的：确认操作、展示状态变化或将注意力引导到新元素。

时长 Token

Token 值 用途
--tabora-duration-fast 120ms 微交互：hover 颜色切换、勾选框切换
--tabora-duration-normal 180ms 标准过渡：弹出框开关、状态切换
--tabora-duration-slow 240ms 进场/退场：弹窗打开、抽屉滑入

缓动函数

Token CSS 值 用途
--tabora-ease-standard cubic-bezier(0.2, 0, 0, 1) 默认缓动。快速启动，温和减速。
--tabora-ease-emphasized cubic-bezier(0.05, 0.7, 0.1, 1) 弹性缓动。用于愉悦时刻：完成任务、添加卡片。

动效演示（hover 触发）

快速 · 120ms · 标准缓动

hover 状态、勾选框切换、图标过渡

标准 · 180ms · 标准缓动

弹出框、展开/折叠、状态切换

缓慢 · 240ms · 强调缓动

弹窗进场、抽屉滑入、全屏打开

动效规则

  • 禁止在 hover 上使用 scale 缩放——引起布局抖动且显得玩具化
  • 禁止连续动画（旋转指示器是唯一例外）
  • 禁止触发重排的属性动画（margin、padding、border-width）
  • 所有动画必须尊重 @media (prefers-reduced-motion: reduce)

图标

图标系统

  • 默认图标库：lucide-solid
  • 默认尺寸：内容区 16px，侧栏/导航区 20px
  • 描边宽度：2px
  • 颜色：继承自父级文本色（currentColor）
使用语义图标

勾选图标表示完成，加号表示添加，齿轮表示配置。图标匹配动作而不是装饰。

用 emoji 做功能图标

emoji 在不同平台上渲染效果不同且无法被主题化。使用 lucide-solid 的 SVG 图标。天气图标可使用自定义 SVG。

可访问性要求

Tabora 面向所有面向用户的功能，目标为 WCAG 2.2 AA 级合规。

最低要求

要求 WCAG 标准 实现方式
色彩对比（普通文本） 1.4.3 (AA) — 4.5:1 所有文本 token 已验证在表面色上的对比度
非文本对比 1.4.11 (AA) — 3:1 边框、图标、焦点环已验证在相邻色上的对比度
键盘可操作 2.1.1 所有可交互元素可聚焦并可通过键盘操作
焦点可见 2.4.7 (AA) 所有元素上的双环焦点指示器（outline + box-shadow）
焦点顺序 2.4.3 逻辑 tab 顺序：侧栏 → 搜索 → 卡片（行优先）→ 设置
标签/说明 3.3.2 所有表单输入有可见标签或 aria-label。图标按钮有 aria-label。
错误识别 3.3.1 错误使用 图标 + 文本 + 颜色。绝不单独使用颜色。
动效控制 2.3.3 尊重 prefers-reduced-motion。无自动播放动画。
触控目标尺寸 2.5.8 (AA) 交互目标 ≥ 24×24px。更小的目标有足够的间距。

焦点环规范

:focus-visible {
  outline: 2px solid var(--tabora-color-focus);
  outline-offset: 2px;
  box-shadow: 0 0 0 4px rgba(26, 144, 112, 0.18);
}

焦点管理

  • 弹窗打开：焦点移至弹窗内第一个可聚焦元素
  • 弹窗关闭：焦点返回至打开弹窗的元素
  • 设置宿主：同弹窗模式
  • 卡片拖拽：必须提供键盘替代方式（上/下/左/右移动）
  • 搜索：方向键导航结果，Enter 选择，Escape 关闭
