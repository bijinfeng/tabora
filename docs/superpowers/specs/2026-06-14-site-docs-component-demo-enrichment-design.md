# Site Docs 组件示例增强设计

日期：2026-06-14

状态：draft

## 目标

增强 `apps/site` 文档页中已经接入真实渲染的组件示例，让示例从“最小可运行片段”提升为“贴近 Tabora 实际使用语境的组合片段”。

本轮只扩展现有真实示例，不新增新的文档路由、组件分类或组件 API。

## 范围

本轮优先补强以下组件示例：

- `Input`
- `Select`
- `Dialog`
- `Table`
- `Toast`

示例源码位于 `packages/ui/src/styled/*/*.demo.tsx`，文档站通过 `apps/site/src/routes/docs/docsExamples.tsx` 直接复用这些 demo 进行真实渲染和源码展示。

## 设计原则

- 示例必须像真实工作台片段，而不是孤立控件陈列。
- 每个示例至少展示 2-3 个关键状态或交互分支。
- 优先展示组件组合方式，例如 `Field + Input + InlineError`、`Table + Badge`、`Dialog + Button`。
- 保持单个 demo 紧凑，不把一个示例做成完整页面。
- 遵守 `DESIGN.md` 中对内容区组件的要求：稳定尺寸、语义控件、可读文本、无宿主级容器越界。

## 组件方案

### Input

改为“工作区搜索设置”片段：

- 展示标签、说明文案、输入值和错误态。
- 展示保存后预览摘要。
- 体现 `Field`、`Input`、`InlineError` 的组合关系。

### Select

改为“默认搜索源配置”片段：

- 展示默认搜索源选择。
- 展示打开方式或附加模式选择。
- 底部同步展示当前配置摘要，证明选择结果会驱动内容变化。

### Dialog

改为“移除插件”危险确认片段：

- 展示触发按钮。
- 展示是否清理本地数据的勾选项或附加说明。
- 展示确认前、确认中、确认后的关键状态之一。

### Table

改为“插件运行状态表”片段：

- 列中同时出现名称、类型、状态、最近活动或权限摘要等信息。
- 至少一列组合 `Badge` 或状态文本，体现密集信息排布。
- 允许通过轻量筛选切换到空状态或另一组行数据，但不做复杂表格功能。

### Toast

改为“设置保存与同步反馈”片段：

- 通过按钮触发不同优先级通知。
- 至少包含一个 action 型 toast。
- 同时体现成功、警告或错误之间的文案差异。

## 测试与验证

先补测试，再改实现。

测试层级：

- 更新 `apps/site` 现有 docs example 测试，验证关键 demo 的源码中包含新的上下文结构和关键交互文案。
- 如有必要，为相关 demo 所在 package 增加聚焦测试，避免只靠快照式检查。

交付前验证：

- `pnpm test`
- `pnpm check`
- 如变更只落在文档站与 UI demo，至少补充 `apps/site` 构建验证。
- 启动本地站点，浏览器检查 `/docs/input`、`/docs/select`、`/docs/dialog`、`/docs/table`、`/docs/toast`。

## 非目标

- 不新增组件种类。
- 不修改 `@tabora/ui` 公共组件 API，除非实现 demo 所必须且有测试覆盖。
- 不重做文档站布局、侧栏结构或组件元数据分类。
